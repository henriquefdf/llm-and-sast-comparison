# Security Analysis Pipeline

This repository contains a comprehensive pipeline for evaluating the performance of Static Application Security Testing (SAST) tools and Large Language Models (LLMs) against the OWASP Benchmark.

## Project Structure

The project is organized into three main directories:

### 1. `pipeline/`

This directory contains the core infrastructure for running the security tools.

- **Purpose**: Automates the execution of CodeQL, Semgrep, Gemini, and GPT against the test cases.
- **Documentation**:
  - [Pipeline README](pipeline/README.md)
  - [Usage Guide](pipeline/USAGE.md)
- **Key Components**:
  - Setup scripts (`setup.sh`) to install dependencies and tools.
  - Execution scripts (`run_pipeline.py`, `run.sh`) to orchestrate the scans.
  - Tool configurations and helper utilities.

### 2. `analysis_pipeline/`

This directory contains the logic for processing and interpreting the raw results.

- **Purpose**: Calculates performance metrics and generates visual reports.
- **Key Features**:
  - **Metrics Calculation**: Computes Precision, Recall, F1-Score, and Accuracy for each tool.
  - **Grouping**: Analyzes performance by CWE (Common Weakness Enumeration) and vulnerability groups (e.g., SQL Injection, XSS).
  - **Visualization**: Generates charts and heatmaps (saved in `output/`) to compare tool performance.
  - **Reporting**: Produces a text-based summary report (`reports.txt`).

### 3. `results/`

This directory serves as the storage for the output generated by the pipeline.

- **Contents**: Raw results, CSV files, and logs from CodeQL, Semgrep, Gemini, and GPT.
- **Note**: This directory is excluded from version control to keep the repository clean.

## Getting Started

To analyze the results:

1. Navigate to the analysis directory:

   ```bash
   cd analysis_pipeline
   ```

2. Run the analysis script:

   ```bash
   python analyze_results.py
   ```

3. Check the `output/` directory for plots and `reports.txt` for the summary.
